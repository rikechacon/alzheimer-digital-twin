Documentación Técnica

Arquitectura del sistema, API y modelo matemático subyacente.

1. Arquitectura del Sistema

El proyecto utiliza un enfoque moderno de separación de responsabilidades:

  • Backend (FastAPI): Gestiona las rutas, la API RESTful y el procesamiento concurrente.
  • Motor Científico (SciPy/NumPy): Resuelve ecuaciones diferenciales ordinarias (EDO) que simulan la dinámica de proteostasis.
  • Frontend (HTML/JS): Interfaz interactiva que se comunica de forma asíncrona mediante Fetch API.

2. Modelo Biomatemático (Proteostasis)

El núcleo de la simulación se basa en un modelo de reacción-difusión (Reaction-Diffusion). Para la proteína Tau, la ecuación se simplifica como:

d_tau/dt = k_tau * tau + coupling * (Graph_Matrix · tau) - clearance * tau

Donde el factor de acoplamiento (coupling) simula la resistencia en las sinapsis, evitando crecimientos exponenciales irreales (overflow), y el clearance (tasa de eliminación) está modulado por el genotipo APOE del paciente.

3. API REST y Endpoints

FastAPI genera automáticamente documentación interactiva (Swagger UI) basada en los modelos Pydantic definidos en main.py. Allí puedes probar simulaciones enviando JSONs directamente al servidor sin pasar por el Dashboard.
